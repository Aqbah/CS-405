# CS-405: Secure Software Development

Welcome to the **CS-405** repository — a collection of assignments, labs, projects, and documentation completed as part of the CS-405 course on *Secure Software Development*.

This repo showcases hands-on work covering common software security vulnerabilities, secure coding practices, encryption, and security policy documentation. Each module’s work demonstrates practical understanding and application of secure coding concepts.

---

## 📚 Course Overview

**CS-405** is a course focused on developing secure software by understanding:

- Common software vulnerabilities (e.g., buffer overflows, SQL injection)
- Secure coding and testing practices
- Encryption fundamentals
- Security policies and documentation
- Risk analysis and mitigation

This repository contains my submitted work for each module and assignment throughout the course.

## 🧠 Module Highlights

### ✅ Module 2 – Common Vulnerabilities
- **Buffer Overflow**  
  Practical demonstration of stack and heap overflow cases.
  
- **SQL Injection**  
  Examples of vulnerable and secured SQL queries with mitigation notes.

### 🔐 Module 5 – Encryption
- Encryption exercises involving cryptographic algorithms.
- Demonstrates secure data handling and cipher implementations.

### 📊 Project Two – Presentation
- **Presentation slides:** Security analysis and design for a real-world scenario.
- Emphasis on identifying threats and proposing security architecture.

### 📝 Security Policy Documentation
- A formal **security policy document** created for a hypothetical organization.
- Defines rules, standards, and practices for secure operations.

### 📘 Module 8 – Journal
- Reflective journal covering advanced secure development concepts.
- Includes insights, learning outcomes, and future recommendations.

### 🧪 Additional Labs and Code
- **Numeric Overflows** – Edge case handling and safe arithmetic coding.
- **Questionable Code** – Code review assignment identifying insecure patterns.
- **Security Unit Tests** – Test cases illustrating secure testing strategies.
- **Exceptions** – Practice with secure exception and error handling.
- **test** – Supplemental test code and scripts.
